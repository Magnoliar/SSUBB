# SSUBB 开发路线图 (Roadmap)

本项目致力于解决"算力与存储分离场景下的全自动字幕补充"问题。以下是宏观版本演进路线。

---

## 已完成版本

### V0.1：核心跑通
- 实现 Coordinator(NAS) / Worker(GPU) 双节点物理拆分。
- NAS 侧完成音频提取、文件上传、字幕落盘闭环。
- Worker 侧集成 Whisper 转写与大语言模型翻译链路。
- 采用 SQLite 持久化任务，确保重启不丢失。

### V0.2 & V0.3：容错与自动化
- **多阶段重试机制**：针对上传、转写、翻译等阶段实现独立的失败重试与超时重置。
- **MoviePilot 集成**：提供专属插件，实现入库即触发的被动全自动化。
- **主动扫描自动化**：内置定时目录扫描器，实现历史遗留无字幕媒体的主动检测与补齐。

### V0.4：质量跃升与交互
- **智能排版系统**：实现分片翻译、时轴矫正与单/双语合并，支持高级 ASS 特效输出。
- **防截断策略**：结合 AI 返回特征进行上下文拼接，解决长句翻译断层问题。
- **极客风 WebUI**：提供暗黑玻璃态实时大盘控制台，清晰展示排队、活跃、失败任务。

### V0.5：零配置工程化
- **Setup Wizard**：双端引入 CLI 与 Web 交互式初始配置向导，彻底告别 YAML 报错。
- **一键 Docker 部署**：Coordinator 端实现免挂载配置文件、内存热启动调度引擎的零门槛体验。
- **自动环境检测**：Worker 端集成 Python / FFmpeg / CUDA 以及 API 联通性开机检测。

### V0.6：性能与调度优化
- **多节点多并发**：支持在局域网内注册多个 GPU Worker，Coordinator 根据 Worker 权重/闲忙状态分发任务。
- **长视频流水线**：串行处理升级为 `音频提取 → A 转写 → A 翻译 / 提取 → B 转写` 的重叠流水线。
- **WebUI 深度管理**：允许在网页上直接控制 Worker 节点上下线，以及配置参数的可视化热修改。

### V0.7：零配置体验 & WebUI 体验升级
- **Toast 通知系统**：任务提交、Worker 上下线、配置保存等操作实时反馈。
- **确认对话框**：危险操作（下线 Worker、关闭自动化）增加二次确认。
- **简单/高级模式**：默认展示精简视图，高级模式显示完整技术细节和实时日志。
- **WebSocket 实时日志**：日志面板通过 WebSocket 实时推送，替代轮询。
- **垂直时间线**：高级模式下任务详情展示阶段耗时的垂直时间线。
- **局域网自动发现**：Coordinator 和 Worker 通过 UDP 广播互相发现，免去手动填写 IP。
- **配置健康度**：Header 显示配置完成度百分比和改进建议。
- **移动端适配**：任务列表在小屏幕下自动变为卡片布局，触控目标优化。
- **无障碍优化**：ARIA 属性、键盘导航、Escape 关闭模态框。

---

### V0.8：智能调度 & 开放接口
- **任务优先级队列**：支持 1-5 级优先级，高优先级任务优先被分发处理。
- **自适应权重**：根据 Worker 历史处理性能自动调整调度权重，高性能节点获得更多任务。
- **自动故障迁移**：Worker 离线后，其正在处理的任务自动迁移到其他可用节点。
- **字幕质量评分**：对生成的字幕进行 0-100 分自动评估（覆盖率、密度、连续性等），低分自动重试。
- **通用 Webhook 接口**：标准化 `/api/webhook` 端点，支持任意外部系统触发任务。
- **API 文档站点**：FastAPI 自动生成的 Swagger UI (`/docs`) 和 ReDoc (`/redoc`)。
- **路由标签**：所有 API 按功能分组（任务管理、Worker 管理、自动化、Webhook 等）。

---

### V0.9：可靠性修复 & 体验打磨 & 批量操作
- **关键 Bug 修复**：时区错误（调度器 UTC+8 窗口错位）、SQL 注入修复（参数化查询）、翻译失败静默返回原文（现在正确触发重试）、异常吞没修复（`_safe_background_task` 包装所有 fire-and-forget 任务）、惊群效应修复（移除 `_watch_loop` 批量重试 pending）。
- **Worker 可靠性**：Whisper 模型缓存（模块级单例，避免每任务 30-60s 重载）、翻译失败检测与 partial 标记、httpx 客户端生命周期管理（连接泄漏修复）、临时文件自动清理、取消端点真实实现。
- **WebUI 体验打磨**：Escape 键关闭所有弹窗、Windows 路径反斜杠修复、API 错误详情 Toast 展示、任务列表分页（每页 20 条）、确认弹窗焦点陷阱与键盘支持、加载状态指示。
- **批量操作**：任务列表 checkbox 多选 + 全选，浮动操作栏批量重试/取消/删除，后端 `POST /api/tasks/batch/{retry,cancel,delete}`。
- **数据洞察**：`GET /api/statistics` 返回 30 天任务趋势、成功率、平均耗时；WebUI 设置面板新增「数据洞察」tab，含 SVG 柱状图和 Worker 利用率卡片。
- **安全加固**：`/api/fs` 路径遍历防护、Worker 文件名净化、配置更新并发锁（`asyncio.Lock`）。
- **心跳并行化**：WorkerRegistry 心跳检查从串行改为 `asyncio.gather()` 并行。

## 当前开发重点

### V0.10：LLM 多源容灾 & 字幕编辑 & 看板重构
- **LLM 多端点容灾**：支持配置多个 LLM 提供商（DeepSeek/OpenAI/智谱等），按优先级自动切换，单点故障不再影响全局。Worker 端 `/api/llm/health` 暴露各提供商健康状态。
- **字幕预览与编辑**：完成任务后可在 WebUI 预览 SRT 字幕内容，支持手动编辑保存、勾选段落重新调用 LLM 优化。
- **字幕命名规范化**：采用 ISO 639-2/B 三字母语言码（chi/eng/jpn），`.ssubb` 标识，`.forced` 标记双语，双语模式强制 ASS 格式。
- **反思翻译**：`need_reflect` 配置项正式生效，翻译完成后可选二次审校提升质量。
- **断句优化增强**：`max_word_count_cjk/english` 配置项正式生效，LLM 优化时遵守行长约束。
- **自动段落级修复**：质量评分 < 60 分时自动扫描异常段落，带滑动上下文窗口定点修复。
- **打分失败保护**：LLM 全部不可用导致评分失败时，不保存字幕，任务标记待重试。
- **配置单入口**：用户只在 Coordinator WebUI 配置 LLM/翻译/优化参数，Coordinator 自动推送到所有在线 Worker 并热重载。
- **扫描历史持久化**：自动扫描结果写入 SQLite，重启不丢失，WebUI 可查看最近扫描记录。
- **看板健康条**：4 个统计卡片替换为紧凑的系统健康条（Coordinator/Worker/LLM 状态一目了然）。
- **任务搜索**：任务列表工具栏新增搜索框，实时过滤影片名称。
- **Worker 详情弹窗**：点击 Worker 卡片查看 GPU 利用率、显存、运行时间、LLM 提供商状态。
- **键盘快捷键**：Ctrl+F 聚焦搜索，Ctrl+Enter 提交任务，Escape 关闭弹窗。

### V0.11：生产就绪（计划中）
- **通知系统**：任务完成/失败时通过 Webhook 推送通知（MoviePilot 已覆盖部分需求）。
- **API 鉴权**：Coordinator API Token 认证，防止未授权访问。
- **日志持久化**：结构化日志写入文件，支持按日期轮转。
- **ASS 样式自定义**：ASS 字幕的字体、大小、颜色、位置可配置。
- **多语言目标翻译**：一次转写后同时生成多种语言字幕。

## 备选需求（Maybe / Someday）

以下功能有讨论价值，但当前优先级较低，视社区反馈决定是否纳入主线：

- **P2P 分布式架构**：去掉中心 Coordinator，多个 SSUBB 实例自行协商并行处理。技术复杂度高，当前的中心化架构已能满足大多数场景。
- **GPU 资源市场**：允许闲置 GPU 节点对外提供付费转写服务。
