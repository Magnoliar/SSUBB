# SSUBB 开发路线图 (Roadmap)

本项目致力于解决“算力与存储分离场景下的全自动字幕补充”问题。以下是宏观版本演进路线。

## 🎯 V0.1：核心跑通（已完成）
- 实现 Coordinator(NAS) / Worker(GPU) 双节点物理拆分。
- NAS 侧完成音频提取、文件上传、字幕落盘闭环。
- Worker 侧集成 Whisper 转写与大语言模型翻译链路。
- 采用 SQLite 持久化任务，确保重启不丢失。

## 🎯 V0.2 & V0.3：容错与自动化（已完成）
- **多阶段重试机制**：针对上传、转写、翻译等阶段实现独立的失败重试与超时重置。
- **MoviePilot 集成**：提供专属插件，实现入库即触发的被动全自动化。
- **主动扫描自动化**：内置定时目录扫描器，实现历史遗留无字幕媒体的主动检测与补齐。

## 🎯 V0.4：质量跃升与交互（已完成）
- **智能排版系统**：实现分片翻译、时轴矫正与单/双语合并，支持高级 ASS 特效输出。
- **防截断策略**：结合 AI 返回特征进行上下文拼接，解决长句翻译断层问题。
- **极客风 WebUI**：提供暗黑玻璃态实时大盘控制台，清晰展示排队、活跃、失败任务。

## 🎯 V0.5：零配置工程化（已完成）
- **Setup Wizard**：双端引入 CLI 与 Web 交互式初始配置向导，彻底告别 YAML 报错。
- **一键 Docker 部署**：Coordinator 端实现免挂载配置文件、内存热启动调度引擎的零门槛体验。
- **自动环境检测**：Worker 端集成 Python / FFmpeg / CUDA 以及 API 联通性开机检测。

---

## 🚀 下一步：V0.6（性能与调度优化，计划中）
- **多节点多并发**：支持在局域网内注册多个 GPU Worker，Coordinator 根据 Worker 权重/闲忙状态分发任务。
- **长视频流水线**：目前的串行处理可升级为 `音频提取 -> A 转写 -> A 翻译 / 提取 -> B 转写` 的重叠流水线。
- **WebUI 深度管理**：允许在网页上直接控制 Worker 节点上下线，以及配置参数的可视化热修改。

## 🚀 远期目标：V1.0
- **开箱即用的跨平台二进制文件 (PyInstaller/Nuitka)**：彻底消除环境安装门槛。
- **P2P 分布式架构**：无需中心节点，多个安装了 SSUBB 的电脑可自行协商并切片处理庞大的视频库。
